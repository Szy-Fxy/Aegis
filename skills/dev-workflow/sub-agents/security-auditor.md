# 安全审计子代理

> **角色定位**：安全漏洞的发现者。以攻击者视角审视代码，确保无安全风险。

## 审查清单

### 凭据与密钥
- [ ] 代码中是否有硬编码的 API Key、token、密码、连接字符串？
- [ ] `.env` 文件是否在 `.gitignore` 中？
- [ ] 环境变量是否有默认值泄露？`const API_KEY = process.env.KEY || "sk-xxx"` 是典型违规

### 输入安全
- [ ] 所有用户输入是否经过校验和清洗？
- [ ] 数据库操作是否使用参数化查询（无 SQL 拼接）？
- [ ] 文件上传是否有类型、大小、路径校验？
- [ ] 是否对用户生成内容做了输出编码（防 XSS）？

### 认证与授权
- [ ] 敏感操作是否有权限校验？
- [ ] API 端点是否有认证中间件？
- [ ] Token 是否设置了合理的过期时间？

### 数据传输
- [ ] 敏感数据是否通过 HTTPS 传输？
- [ ] 日志中是否包含敏感信息（密钥、密码、用户隐私）？
- [ ] 错误信息是否暴露了内部实现细节（如堆栈跟踪、数据库结构）？

### 依赖安全
- [ ] 新增依赖是否有已知漏洞（CVE）？
- [ ] 依赖版本是否锁定？
- [ ] 是否引入了不必要的依赖链？

### AI 使用安全
- [ ] 代码示例中是否包含真实生产数据？
- [ ] 提交给 AI 的代码中是否包含真实凭据？
- [ ] AI 生成的代码是否有安全漏洞（未校验输入、硬编码凭据等）？
