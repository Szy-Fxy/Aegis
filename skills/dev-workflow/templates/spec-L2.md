# L2 方案设计模板

> L2 阶段合并文档：一个文件搞定 brainstorm + proposal + design + spec。

```markdown
# {功能名称} — 方案设计

## 背景
[为什么有这个需求？1-3 句话]

## 目标
- [目标 1]
- [目标 2]

## 非目标
- [本次不做什么]

## 问题
[当前痛点，1-2 句话]

## 方案
[选定的方案 + 为什么选它]

## 变更概要
- **新增** / **修改** / **删除** [列出]

## 核心接口
[新 API 签名，非必须则不写]

## 验收场景 (Given-When-Then)
### 场景: {场景名}
- **当** {条件} → **则** {预期结果}

### 场景: {场景名}
- **当** {边界/异常条件} → **则** {预期结果}

## 方案风险边界
| 什么情况下这个方案会失败？ | 为什么？ |
|--------------------------|----------|
| {场景 1} | {原因} |
| {场景 2} | {原因} |

### 已知限制
- {方案明确不覆盖的场景}
- {方案不适用的极端情况}

## 风险与备注
- [潜在风险及缓解措施]
```

**规则**：
- 一个文件搞定，不拆多个
- 背景/目标/非目标 防止范围蔓延
- 验收场景至少覆盖正常路径 + 1 个边界条件
- 方案风险边界至少列出 2 个风险场景，不可为空
