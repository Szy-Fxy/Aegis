# DevLogs — 开发日志存档

> 功能完成后 AI 自动写入的开发日志。命名格式见 `conventions/naming-and-formats.md`。
>
> 进程恢复：AI 在每次对话开始时读取最新 DevLog 恢复上下文。

## DevLog 模板

```markdown
# {项目名} — {功能描述}

## 元信息

| 字段 | 值 |
|------|-----|
| 需求级别 | L1 / L2 / L3 |
| 开始日期 | YYYY-MM-DD |
| 完成日期 | YYYY-MM-DD |
| 关联需求 | REQ-XXX |

## 当前进度

[一句话描述当前状态，例如：L3-4 spec 编写中，待用户审核]

## 下一动作

[用户审核 → 进入阶段 N / 等待用户确认 / 已完成]

## 步骤进度

| 步骤 | 状态 | 备注 |
|------|------|------|
| 1. 头脑风暴 | ✅完成 | 3 个方案，选定方案 B |
| 2. 提案 | ✅完成 | 用户已确认 |
| 3. 技术设计 | 🔨进行中 | API 设计已完成，数据流待补充 |
| 4. 需求规格 | ⏸️待开始 | — |
| 5. 任务拆分 | ⏸️待开始 | — |
| 6. 集成审核 | ⏸️待开始 | — |
| 7. 实现验证 | ⏸️待开始 | — |

## 关键决策

- [决策 1]：[原因]
- [决策 2]：[原因]

## 改动清单

| 文件 | 操作 | 说明 |
|------|------|------|
| path/to/file | 新增/修改/删除 | 一句话说明 |

## 提交记录

| Commit | 说明 |
|--------|------|
| `abc123` | 初始提交 |

## 教训

- [从本次需求中学到的经验]
- [踩过的坑及避免方法]
- [5 维审视结果]
```

## 状态值

| 状态 | 含义 |
|------|------|
| ✅完成 | 已完成并通过 |
| 🔨进行中 | 正在执行 |
| ⏸️暂停 | 等待外部条件 |
| ⏸️待开始 | 尚未开始 |
| ❌取消 | 不再继续 |

## 记录时机

- **L1**：可选（学到新东西或踩了坑时写）
- **L2**：完成后必须记录
- **L3**：每个批次完成后追加，最终结束时提交完整日志
