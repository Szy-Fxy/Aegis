# 05-tasks 模板

> L3-5 阶段：将 spec 分解为可逐个完成的任务。

```markdown
# 任务清单

## 验收标准对照（从 02-proposal + 04-spec 摘录）

| # | 验收标准 | 来源 | 对应任务 |
|---|----------|------|----------|
| 1 | {标准描述} | 02-proposal | Task 1 |
| 2 | {标准描述} | 04-spec | Task 3 |

## 1. {模块名} — {操作类型（新增/修改/重构）}
- [ ] 1.1 {具体操作描述（≤3 文件、单一操作类型）}
  - 验收条件：{从上方验收标准对照表引用}
- [ ] 1.2 {具体操作描述}
  - 验收条件：{同上}

## 2. {模块名} — {操作类型}
- [ ] 2.1 {具体操作描述}
  - 验收条件：{同上}

## N. 编译/测试验证
- [ ] N.1 {验证命令或步骤}
- [ ] N.2 {验证命令或步骤}
```

**规则**：
- 每个任务 ≤3 文件、单一操作类型（新增/修改/删除一次只一种）
- 任务按依赖关系排序，前面的完成后面才能开始
- 最后一项永远是验证（编译/lint/test）
- **每个任务必须附带验收条件**：从 02-proposal 和 04-spec 中摘录，写到该任务下方
- **顶部必须有「验收标准对照」表**：确保每个验收标准都能追溯到具体任务
