# 05-tasks 模板

> L3-5 阶段：将 spec 分解为可逐个完成的任务。

```markdown
# 任务清单

## 1. {模块名} — {操作类型（新增/修改/重构）}
- [ ] 1.1 {具体操作描述（≤3 文件、单一操作类型）}
- [ ] 1.2 {具体操作描述}

## 2. {模块名} — {操作类型}
- [ ] 2.1 {具体操作描述}

## N. 编译/测试验证
- [ ] N.1 {验证命令或步骤}
- [ ] N.2 {验证命令或步骤}
```

**规则**：
- 每个任务 ≤3 文件、单一操作类型（新增/修改/删除一次只一种）
- 任务按依赖关系排序，前面的完成后面才能开始
- 最后一项永远是验证（编译/lint/test）
