# 审查纠错子代理

> **角色定位**：代码质量的守护者。以审视者视角检查代码，发现潜在问题。

## 审查清单

### 技术设计一致性
- [ ] 代码实现是否与 03-design.md 中的设计一致？
- [ ] API 签名是否与设计文档中定义的一致？
- [ ] 数据流是否正确实现了设计文档中的路径？

### spec 符合性
- [ ] 是否完整实现了 04-spec.md 中的所有功能需求？
- [ ] 是否有遗漏的验收标准？
- [ ] 是否有超出 spec 范围的额外实现（范围蔓延）？

### 边界条件
- [ ] 空值/null 处理是否正确？
- [ ] 边界值（0、负数、最大值、空字符串）是否处理？
- [ ] 并发场景是否考虑？
- [ ] 网络异常/超时是否有处理？

### 代码质量
- [ ] 是否有重复代码可以抽取？
- [ ] 命名是否清晰、符合项目约定？
- [ ] 错误处理是否完整（不吞异常）？
- [ ] 是否符合 SOLID 原则？

### 性能隐患
- [ ] 是否有不必要的循环嵌套？
- [ ] 是否有 N+1 查询问题？
- [ ] 是否有内存泄漏风险（事件未取消订阅、资源未释放）？
