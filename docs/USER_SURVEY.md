## 二、流程执行检查（必填）

### L1 快速修复

- [ ] 修改前更新了 INDEX.md？
- [ ] 修改后展示了改动摘要（文件路径 + 增删行数 + 改动点简述）？
- [ ] 等待了用户确认？
- [ ] 修改后写了 DevLog？
- [ ] INDEX.md 状态更新为 ✅ done？

### L2 标准需求

#### L2-1 方案设计
- [ ] 创建了 `Aegis_Specs/L2/{feature-name}/design.md`？
- [ ] 向用户展示了方案内容？
- [ ] 等待了用户明确确认（如"OK"、"可以"）？
- [ ] **BOUNDARY CHECK**：用 `read` 工具确认 design.md 存在？

#### L2-2 实现
- [ ] 按 design.md 写代码？
- [ ] 运行了 typecheck / lint / build？
- [ ] 展示了改动摘要（文件路径 + 增删行数 + 改动点简述）？
- [ ] 没有擅自执行 git/svn 操作？
- [ ] 等待了用户确认？
- [ ] **BOUNDARY CHECK**：代码编译通过？

#### L2-3 验证收尾
- [ ] 更新了 INDEX.md（标记 ✅ done）？
- [ ] 写了 DevLog？
- [ ] 告知用户完成？
- [ ] **BOUNDARY CHECK**：INDEX.md 已更新 + DevLog 已写入？

### L3 大型重构

- [ ] 01-brainstorm.md 存在？
- [ ] 02-proposal.md 存在？
- [ ] 03-design.md 存在？
- [ ] 04-spec.md 存在？

- [ ] 07-verify.md 存在？
