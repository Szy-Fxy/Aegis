# 06-review 模板

> L3-6 阶段：对所有已产出文档做一次完整性审核。

```markdown
# 集成审核

## 文档完整性
| 文档 | 状态 | 备注 |
|------|------|------|
| 00-MetaSpec.md | ✅/❌/— | （— 表示无此文档） |
| 01-brainstorm.md | ✅/❌ | |
| 02-proposal.md | ✅/❌ | |
| 03-design.md | ✅/❌ | |
| 04-spec.md | ✅/❌ | |
| 05-tasks.md | ✅/❌ | |

## 交叉检查
1. 04-spec 的场景是否全部被 05-tasks 覆盖？
2. 04-spec 是否违反了 00-MetaSpec 的约束？
3. 03-design 的 API 是否满足 04-spec 的场景需求？
4. 各文档之间的术语是否一致？

## 审核结论
- [ ] 全部通过，进入执行（阶段 7）
- [ ] 需修改（详见备注），修改后重新审核
```

**规则**：
- 「交叉检查」是防止文档之间脱节的关键步骤
- 如果缺少某个文档（如无 00-MetaSpec），标记为「—」而不是❌
