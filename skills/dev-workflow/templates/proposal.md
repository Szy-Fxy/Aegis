# 02-proposal 模板

> L3-2 阶段：选定方案、列出变更概要、评估影响范围。

```markdown
# 提案：{功能名称}

## 为什么
[业务价值 / 解决的问题 / 不做的后果]

## 选定方案
[基于 brainstorm，选哪个方案 + 为什么]

## 变更概要
1. **新增** {API/文件/类名} — {一句话说明}
2. **修改** {API/文件/类名} — {一句话说明}
3. **删除** {API/文件/类名} — {一句话说明}

## 影响范围
| 文件 | 操作 |
|------|------|
| path/to/file.h | 新增/修改/删除 |

## 不在本次范围
- [明确排除的内容]
```

**规则**：
- 影响范围 >10 个文件时考虑拆分提案
- 「不在本次范围」是防御性的，防止范围蔓延
