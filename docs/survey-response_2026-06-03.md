## 五、改进建议（AI 视角）

### 基于本次任务，Aegis 规则哪里需要改进？

1. **BOUNDARY CHECK 增加"顺序核查"**：不仅检查文件存在，还要检查"design.md 的创建时间是否早于代码修改时间"。可以用 git log 或文件时间戳来交叉验证。
2. **增加"模板保护"规则**：模板类文件（如 USER_SURVEY.md、spec-L2.md）必须保持空白/模板状态，AI 不得直接修改，应新建副本填写。
3. **沙箱豁免规则**：当 vite build / dev server 因环境限制无法运行时，应在 DevLog 中记录"未运行原因 + 等效验证方式"，并告知用户。

### 本次任务中，哪些规则特别有效？哪些完全没用？

- **特别有效的规则**：BOUNDARY CHECK 的 read 确认（否则 WS-008 design.md 根本不会创建）。DevLog 迫使 AI 复盘错误。
- **完全没用的规则**：没有完全没用的规则。L2-1"展示方案等确认"有效，是 AI 没执行。
