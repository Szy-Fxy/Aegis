# TempData — 临时参考资料

> 存放开发过程中临时查找的资料，经审查代理验证后暂存于此。
>
> 命名格式和完整规范见 `Aegis/skills/dev-workflow/conventions/naming-and-formats.md`。

## 提炼到 TechStack

功能开发中如果 AI 发现 TechStack 缺少某块技术知识：
1. 网上搜索 → 审查验证 → 暂存 TempData
2. 使用该知识完成功能
3. 功能完成后，提炼核心要点精简写入 TechStack

**要点**：写入 TechStack 的是广泛通用知识（如「对象池」），不是具体实现（如「子弹对象池」）。保持 TechStack 精简。

## 核心原则

- 技术栈文件定位为「引子」（广而泛），不要求专精
- 当本地 TechStack 不够用时 → 网上查 → 放 TempData → 用完后提炼精华合并回 TechStack
